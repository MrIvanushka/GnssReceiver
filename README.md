# GNSS Receiver

`GNSS Receiver` - это набор библиотек, предназначенных для построения модели навигационного приёмника. На данный момент, в репозитории присутствует только парсер навигационных сообщений, но со временем он обрастёт и другими модулями.

## Список библиотек

### CommonUtilities

`CommonUtilities` - библиотека, содержащая утилиты, общие для прочих проектов. В ней содержатся логгер и класс статистики.

### FrameParser

`FrameParser` принимает на вход набор отчётов (мнимую часть), полученных в результате трекинга сигналов навигационных спутников, производит синхронизацию бит, подкадров и выделяет полезную информацию. На данный момент `FrameParser` способен обрабатывать только данные GPS и читать только оперативную информацию (все типы сообщений, кроме альманаха).

### Math

`Math` - библиотека, содержащая классы, утилиты и переопределения, необходимые для решения СЛАУ, а также представления координат объекта в различных декартовых системах.

### ReceiverLocationEstimator

`ReceiverLocationEstimator` расчитывает координаты и корректирует показания часов навигационного приёмника на основе псевдозадержки и координат спутника на момент предшествия отправки дальномерного кода. Технически, применим для любой навигационной системы, но на данный момент предоставляется только под GPS и ГЛОНАСС.

### SatLocationEstimator

`SatLocationEstimator` принимает на вход эфемеридные данные спутника на заданный момент времени и предназначен для расчёта его координат на момент предшествия отправки дальномерного кода. Работает с протоколами GPS и ГЛОНАСС.

## Сборка

Варианты использования библиотек описаны в примерах. Библиотеки и примеры собираются как субдиректории. Сборка примеров по умолчанию отключена, но её можно включить с помощью опций cmake.

Для сборки на Windows можно использовать скрипты `Build.bat` или `BuildWithExamples.bat`. Также сборку можно осуществить с помощью командной строки:

```
  $ cd _examples/<example>
  $ mkdir _build
  $ cd _build
  $ cmake -DGNSS_RECEIVER_BUILD_EXAMPLES=<ON/OFF> ../
```

Некоторые примеры могут опираться на txt-файлы с данными. Данные файлы лежат в папке с исходниками примера, их следует переместить в папку со сборкой (в папку с <example>.sln на Windows) или в папку с бинарным файлом (в случае запуска не через IDE). При сборке с помощью скрипта, необходимые файлы копируются автоматически.
